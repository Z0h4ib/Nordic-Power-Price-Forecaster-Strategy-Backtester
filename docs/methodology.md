# Methodology

## 1. Data pipeline

The pipeline is built around deterministic hourly joins across ENTSO-E market data and Open-Meteo weather data. ENTSO-E is queried for day-ahead prices and renewable generation (wind onshore/offshore, solar) for DK1 and DK2. Open-Meteo provides exogenous weather covariates (temperature and wind speed at Danish grid-relevant coordinates). API pulls run in monthly chunks with retry/backoff around transient ENTSO-E failures. All timestamps are normalized to UTC before merge and storage, which removes DST ambiguity and guarantees one canonical hourly key across sources.

Loading is idempotent by design: inserts are conflict-safe and can be rerun without row duplication. Data is staged into PostgreSQL tables for spot prices, generation, weather, engineered features, and backtest outputs, so the same extraction window and code version regenerate identical inputs.

## 2. Feature engineering

Features are explicitly structured to capture electricity-specific seasonality, autoregression, and merit-order effects. The core lag stack includes short-memory lags and weekly seasonality, with lag-168 (same hour last week) being particularly informative for load shape recurrence and recurring intraday dispatch patterns. Rolling moments (mean/std over short and weekly windows) proxy local level and volatility and feed both forecasting and trading layers.

Calendar variables are encoded cyclically rather than ordinally. For hour-of-day and month-of-year, sin/cos encoding preserves circular topology: hour 23 is adjacent to hour 0 in feature space instead of being treated as maximally distant. That improves tree splits around boundary hours and month transitions.

Generation-side features include aggregate wind and a renewables share proxy. In practice, this captures the non-linear price suppression seen in high-wind intervals when marginal thermal generation is displaced. SHAP diagnostics in this project are consistent with that intuition: top contributors are price lags (24h, 168h) and wind-related variables.

## 3. Walk-forward validation

Validation uses an expanding-window walk-forward scheme with strict temporal ordering; no random splits are used. Each fold trains on all available history up to a cutoff and tests on the subsequent 168-hour block. This mirrors a live retraining loop and prevents leakage from future price regimes into training.

The realized fold counts were 5 for DK1 and 33 for DK2. Fold imbalance is data-availability driven, not methodological. Even with fewer DK1 folds, the same protocol is enforced for both zones.

## 4. Model selection

Model selection was anchored on baseline-first discipline. Persistence and Ridge were fit before gradient boosting to quantify incremental value from non-linearity. Average walk-forward MAE (EUR/MWh) was:

- DK1: persistence 39.74, Ridge 74.07, XGBoost 37.38
- DK2: persistence 41.87, Ridge 49.86, XGBoost 35.31

That implies XGBoost MAE improvements over persistence of 5.94% (DK1) and 15.67% (DK2). Ridge underperforming persistence is informative: linear effects alone are insufficient under regime-dependent merit-order dynamics. XGBoost was selected because it handles feature interactions and non-linear thresholds cleanly, while remaining fast enough for repeated walk-forward retraining and interpretable enough (via SHAP) for model risk review.

## 5. Backtesting design

Signals are generated by comparing model-implied value to a forward proxy: a rolling 168-hour mean of realized spot. Position logic is ternary (+1, -1, 0) with an entry threshold to avoid trading weak edges. P&L is computed hourly as signal times spread between realized spot and forward proxy, then aggregated into standard risk metrics.

At threshold = 5%, DK1 delivered total return 10,458.52, Sharpe 16.59, Sortino 21.04, max drawdown -266.21, trade count 649, and win rate 72.27%. DK2 at the same threshold produced return 67,089.72, Sharpe 14.25, Sortino 19.54, max drawdown -767.94, 4,751 trades, and 65.00% win rate. Threshold sensitivity (0%, 2%, 5%, 10%) showed the expected precision-frequency tradeoff: tighter thresholds reduced trade count (720 to 581) while increasing win rate (70.83% to 73.49%) and modestly lowering Sharpe.

## 6. Monte Carlo (price paths and return bootstrap)

Monte Carlo is used for two inference questions.

Phase 3 (price-path uncertainty): residual bootstrap around point forecasts generates percentile bands (p5/p25/p75/p95) over future hours. This answers calibration and interval coverage questions around forecast uncertainty, not strategy robustness.

Phase 4 (return bootstrap): daily strategy returns are resampled to obtain a distribution of risk-adjusted performance. For DK1, 1,000 simulations yielded Sharpe percentiles p5/p50/p95 = 11.4/17.3/25.1. This addresses path dependence directly: observed Sharpe is not treated as a single deterministic point but as one draw from a return distribution.

## 7. Regime dependence

Regime analysis split performance into Energy crisis, Post-crisis, and Stabilization periods. Reported results show all realized edge concentrated in Stabilization: return 10,458.52, Sharpe 16.59, win rate 72.27%, 649 trades, while the earlier two regimes contributed zero realized trades and return under the final threshold logic.

The interpretation is important: the strategy edge is conditional on market state, not universal across structural shocks. For deployment, this argues for explicit regime gating and periodic retraining/retuning rather than static parameters. In other words, walk-forward validation and bootstrap robustness are necessary but not sufficient; regime awareness is a first-class control variable in power markets.
